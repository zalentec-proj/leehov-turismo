# Regra Operacional do Projeto

Este documento define a regra de controle do projeto **Leehov Turismo**.

## Regra principal

Nenhuma ação deve ser feita no GitHub sem aprovação explícita do responsável pelo projeto.

Isso inclui:

1. Criação de arquivos.
2. Alteração de arquivos.
3. Exclusão de arquivos.
4. Commits.
5. Criação de branches.
6. Pull requests.
7. Instalação de dependências.
8. Configuração inicial do projeto.
9. Envio de código.
10. Envio de documentação.
11. Alterações em estrutura de pastas.
12. Mudanças em banco de dados, migrations ou seeds.

## Fluxo obrigatório

Antes de qualquer ação no repositório:

1. Apresentar o que será feito.
2. Listar arquivos afetados.
3. Explicar o objetivo da alteração.
4. Informar a mensagem de commit pretendida, quando houver.
5. Aguardar aprovação explícita.

## Exemplo de aprovação válida

```txt
Aprovado: subir documentos norteadores no repositório zalentec-proj/leehov-turismo
```

## Exemplo de ação proibida

Não executar comandos como `git commit`, `git push`, instalação de dependências, criação de arquivos ou alteração de arquivos apenas com base em intenção implícita.

## Objetivo da regra

Evitar alterações não autorizadas, perda de controle técnico e decisões improvisadas em um projeto que precisa ser modular, documentado e evolutivo.
